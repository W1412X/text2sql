### （一）论文（设计）的主要内容

本课题面向工业场景中的数据质量评估需求，研究如何利用大语言模型（LLM）将工程师的自然语言查询（如“查一下温度字段有没有空值”）自动转化为可执行的 SQL 语句。与通用 NL2SQL 系统不同，本系统聚焦于**数据完整性、准确性、一致性等质量维度**，强调生成结果的**可靠性**与**可落地性**。

具体工作包括：  
1. **构建面向工业数据质量的测试集**：围绕设备监控、传感器日志等典型场景，设计 100+ 条中文自然语言查询及其标准 SQL，覆盖缺失值、异常值、唯一性等常见质量检查任务；  
2. **设计分层处理架构**：简单查询（如单字段空值统计）通过规则模板或轻量蒸馏模型快速生成；复杂或模糊查询（如“最近异常的记录”）交由智能 Agent 处理，支持多轮澄清；  
3. **实现自主 Schema 探索机制**：借鉴 AutoLink 思路，不一次性输入全库结构，而是结合数据库探针与向量检索，动态获取相关表和字段，解决工业库字段命名混乱、规模大的问题；  
4. **开发后处理校验模块**：对生成的 SQL 进行语法、Schema 匹配和安全性检查，拦截幻觉与危险操作；  
5. **搭建评估体系**：从准确率、幻觉率、响应时间、交互次数等维度量化系统性能，并对比不同 LLM（如 Qwen、ChatGLM）在数据质量任务中的表现。

最终目标是交付一个**可演示、可评估、贴近真实工业需求**的原型系统，为一线工程师提供低门槛的数据质量自查工具。

---

### （二）理论意义和应用价值

**理论层面**，本研究探索了大语言模型在**垂直领域结构化任务**中的能力边界。通过引入意图分层、自主 Schema 链接、规则约束等机制，系统性地缓解了 LLM 在 NL2SQL 任务中常见的幻觉、上下文过载和领域术语理解不足等问题，为专业领域的语言模型应用提供实践参考。

**应用层面**，工业现场的数据质量检查往往依赖 DBA 或数据工程师编写 SQL，门槛高、效率低。本系统允许普通操作人员用自然语言直接发起质量评估，**显著降低技术门槛**，加速数据问题发现与治理。同时，通过蒸馏小模型处理高频简单查询，兼顾了响应速度与部署成本，具备实际落地潜力。
